boost::signal

다른 말로 설명해서, Observer Pattern 혹은 Publish/Subscriber Pattern을 말한다.

시그널을 생성시 연관되어질 함수를 템플릿 인자로 넘긴다.
그리고 signal s는 connect()를 이용해서 실제 함수와 연관지어진다.
이후 시그널 s를 통해서 함수 호출이 가능하다.


#include <boost/signal.hpp>
#include <iostream>

void func()
{
   std::cout << "Hello, world!" << std::endl;
}

int main()
{
   boost::signal< void() > s;
   s.connect(func);
   s();
}

그런데 boost::function 을 이용해서도 이것은 가능하다. 즉,

#include <boost/function.hpp>
#include <iostream>

void func()
{
   std::cout << "Hello, world!" << std::endl;
}

int main()
{
   boost::function<void ()> f;
   f = func;
   f();
}

하지만 시그널은 여러개의 함수와 연관지어질수있다. 즉,
#include <boost/signal.hpp>
#include <iostream>

void func1()
{
   std::cout << "Hello" << std::flush;
}

void func2()
{
   std::cout << ", world!" << std::endl;
}

int main()
{
   boost::signal<void ()> s;
   s.connect(func1);
   s.connect(func2);
   s();
}

함수호출순서를 명확하게 지정하고 싶다면 오버로드된 connect()함수를 이용한다.즉,
int main()
{
   boost::signal<void ()> s;
   s.connect(1, func2);
   s.connect(0, func1); // 0 지정, 1로 지정된 func2보다 먼저 수행됨
   s();
}

disconnect()를 이용해서 연관을 해제한다.

int main()
{
   boost::signal<void ()> s;
   s.connect(func1);
   s.connect(func2);
   s.disconnect(func2);
   s();
}

그밖에 num_slots(), empty(), disconnect_all_slots() 함수가 존재함.

만약 연관되는 함수들이 리턴값이 있다면,
#include <boost/signal.hpp>
#include <iostream>

int func1()
{
   return 1;
}

int func2()
{
   return 2;
}

int main()
{
   boost::signal<int ()> s;
   s.connect(func1);
   s.connect(func2);
   std::cout << s() << std::endl;
}
위 예제에서는 func2의 리턴값 2 만이 출력된다.
즉, 모든 함수들중에서 마지막으로 호출된 함수의 리턴값만이 출력된다.

이것을 조정하고 싶다면 operator()()를 재정의하는 클래스를 생성해서 시그널 생성시에 전달해야 한다.
#include <boost/signal.hpp>
#include <iostream>
#include <algorithm>

int func1()
{
   return 0;
}

int func2()
{
   return 1;
}

template <typename T>
struct min_element
{
  typedef T result_type;

  template <typename InputIterator>
  T operator()(InputIterator first, InputIterator last) const
  {
      return * std::min_element(first, last);            
  }
};

int main()
{
   boost::signal<int (), min_element<int> > s;
   s.connect(func1);
   s.connect(func2);   
   std::cout << s() << std::endl; // 0 출력.
}


각 함수들의 리턴값들을 모두 구하는 경우에는 다음처럼 가능.

...
template <typename T>
struct min_element
{
   typedef T result_type;

   template <typename InputIterator>
   T operator()(InputIterator first, InputIterator last) const
   {
     return T(first, last);
   }
};

int main()
{
   boost::signal<int (), min_element<std::vector<int> > > s;
   s.connect(func1);
   s.connect(func2);
   std::vector<int> v = s();
   std::cout << *std::min_element(v.begin(), v.end()) << std::endl;
}

boost::signals::connection

connect()함수는 boost::signals::connection 타입을 리턴한다. 이것을 이용해서
void func()
{
   std::cout << "Hello, world!" << std::endl;
}

int main()
{
   boost::signal<void ()> s;
   boost::signals::connection c = s.connect(func);
   s();
   c.disconnect(); //s.disconnet(func) 대신
}

그밖에 block() 과 unblock()함수 가 있다.
#include <boost/signal.hpp>
#include <iostream>

void func()
{
   std::cout << "Hello, world!" << std::endl;
}

int main()
{
   boost::signal<void ()> s;
   boost::signals::connection c = s.connect(func);
   c.block();
   s(); // 이시그널은 실행 안됨!!
   c.unblock();
   s(); // 이제 실행된다.
}

boost::signals::scoped_connection

소멸자에서 자동으로 connection이 해제된다(scoped).

#include <boost/signal.hpp>
#include <iostream>

void func()
{
  std::cout << "Hello, world!" << std::endl;
}

int main()
{
boost::signal<void ()> s;
{
boost::signals::scoped_connection c = s.connect(func);
}
s(); //호출 안됨. 이미 connection이 해제되었음!
}
scoped_connection은 connection을 상속받았기 때문에 동일한 메소드를 제공한다.


boost::signals::trackable

scoped_connection 없이 자동 해제 기능을 구현하고 싶을때, 즉,

//문제있는 코드
#include <boost/signal.hpp>
#include <boost/bind.hpp>
#include <iostream>
#include <memory>

class world
{
public:
    void hello() const
    {
        std::cout << "Hello, world!" << std::endl;
    }
};

int main()
{
    boost::signal<void ()> s;
   
    {
        std::auto_ptr<world> w(new world());        
        s.connect(boost::bind(&world::hello, w.get()));
    }
    //scope밖, 이 시점에 w는 해제됨.
   
    std::cout << s.num_slots() << std::endl; // w가 해제됬지만 1을 출력 !
   
    s(); // w가 해제됬지만 hello() 함수를 호출시도. 호출된다고 해도 문제발생 소지.
}

문제점을 수정하기 위해서는 간단히 boost::signals::trackable을 상속받는다.
#include <boost/signal.hpp>
#include <boost/bind.hpp>
#include <iostream>
#include <memory>

class world : public boost::signals::trackable
{
  public:
    void hello() const
    {
      std::cout << "Hello, world!" << std::endl;
    }
};

int main()
{
   boost::signal<void ()> s;
   {
     std::auto_ptr<world> w(new world());
     s.connect(boost::bind(&world::hello, w.get()));
   }
   std::cout << s.num_slots() << std::endl; //0을 출력
   s(); //hello 함수 호출 없음.
}

시그널에 함수를 연관시킬때 객체가 아닌, 객체에 대한 포인터가 사용된 경우에
connection관리를 간단하게 해준다.
