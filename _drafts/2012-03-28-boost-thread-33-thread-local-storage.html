Thread Local Storage (TLS)
쓰레드 전용 저장소를 제공. 쓰레드 내부에서만 사용되는 전역변수.

난수 3개를 발생하는 예제로부터 시작한다.


#include <boost/thread.hpp>
#include <iostream>
#include <cstdlib>
#include <ctime>

void init_number_generator()
{
    static bool done = false;
    if (!done)
    {
        done = true;
        std::srand(static_cast<unsigned int>(std::time(0)));
    }
}

boost::mutex mutex;

void random_number_generator()
{
    init_number_generator();
    int i = std::rand();
    boost::lock_guard<boost::mutex> lock(mutex);
    std::cout << i << std::endl;
}

int main()
{
    boost::thread t[3];

    for (int i = 0; i < 3; ++i)
        t[i] = boost::thread(random_number_generator);

    for (int i = 0; i < 3; ++i)
        t[i].join();
}

출력
24264
41
41

이 예제는 첫번째와  2,3번째가 틀린 숫자를 출력하는 문제가 있다.
이는 std::srand()가 한번만 호출되기 때문이다. 각 쓰레드에서 난수값을 초기화
시키고 싶다면, TLS를 사용하여 이를 수정할수 있다.

boost::thread_specific_ptr

#include <boost/thread.hpp>
#include <iostream>
#include <cstdlib>
#include <ctime>

boost::mutex mutex;

void init_number_generator()
{
    static boost::thread_specific_ptr<bool> tls;

    boost::lock_guard<boost::mutex> lock(mutex);

    if (!tls.get())
    {
        tls.reset(new bool(false)); //스레드 종료시에 tls개체에게 delete가 호출된다.
        std::cout << boost::this_thread::get_id()<<" : TLS reset!" << std::endl;
    }

    if (!*tls)
    {
        *tls = true;
        std::srand(static_cast<unsigned int>(std::time(0)));
        std::cout << boost::this_thread::get_id()<<" : std::srand" << std::endl;
    }
}

void random_number_generator()
{
    init_number_generator();
    int i = std::rand();
    boost::lock_guard<boost::mutex> lock(mutex);
    std::cout << boost::this_thread::get_id()<<" : "<<i<< std::endl;
}

int main()
{
    boost::thread t[3];

    for (int i = 0; i < 3; ++i)
        t[i] = boost::thread(random_number_generator);

    for (int i = 0; i < 3; ++i)
        t[i].join();
}

정적변수 done대신에 boost::thread_specific_ptr 클래스 객체가 사용되었다.
결국 done과 동일하게 동작한다. 하지만 tls 객체에 저장된 값은
해당 쓰레드에서만 볼수있고 사용가능하다. tls 에는 변수의 주소를 지정해줘야하는데
reset()메서드를 이용한다. get()을 이용하면 이미 주소가 할당되었는지 알수있다.
boost::thread_specific_ptr는 주소를 저장하고 있기때문에 일반포인터처럼 동작한다.
그결과 operator*() 와 operator->() 연산자들이 간편한 사용을 목적으로 오버로딩되었다.
예제에서는 *tls 로 bool변수를 판단하고 있다.

각 쓰레드마다 난수가 초기화 된다는 사실은 확인 가능하다.
