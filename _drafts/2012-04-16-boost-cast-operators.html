C++표준에서는 static_cast, dynamic_cast, const_cast, reinterpret_cast
4가지의 캐스트 연산자를 제공하고 있다.
Boost.Conversion 와 Boost.NumericConversion 라이브러리는 추가적인 캐스팅 연산자를 제공한다.


 Boost.Conversion 
Boost.Conversion 은 헤더파일 2개로만 구성되어 있다.
boost/cast.hpp 헤더는 boost::polymorphic_cast 와 boost::polymorphic_downcast 를 정의하고,
boost/lexical_cast.hpp 헤더에서는 boost::lexical_cast 를 제공한다.

boost::polymorphic_cast 와 boost::polymorphic_downcast의 목적은
타입캐스트를 좀더 정확하게 구체화 시키기 위해서 이다.
일반적으로 dynamic_cast을 이용해서 처리되는 다음 예제를 보자.

struct father
{
    virtual ~father() { };
};

struct mother
{
    virtual ~mother() { };
};

struct child : public father, public mother
{
};

void func(father *f)
{
    child *c = dynamic_cast<child*>(f);
}

int main()
{
    child *c = new child;
    func(c);

    father *f = new child;
    mother *m = dynamic_cast<mother*>(f);
}

예제에서 dynamic_cast 를 2번 사용했다.
func() 에서 부모클래스의 포인터를 자식 클래스를 가르키는 포인터로 변환시키고 있고,
main()에서는 부모클래스의 포인터를 다른 부모클래스의 포인터로 변환시키고 있다.
첫번째 캐스트는 downcast 로 불리고, 두번째경우는 cross cast로 각각 불린다.
Boost.Conversion의 캐스트 연산자를 이용하면 downcast와 cross cast를 구분할수 있다.

#include <boost/cast.hpp>

struct father
{
   virtual ~father() { };
};

struct mother
{
   virtual ~mother() { };
};

struct child : public father, public mother
{
};

void func(father *f)
{
   child *c = boost::polymorphic_downcast<child*>(f);
}

int main()
{
   child *c = new child;
   func(c);

   father *f = new child;
   mother *m = boost::polymorphic_cast<mother*>(f);
}

boost::polymorphic_downcast는 오직 downcast를 위해서만 사용된다.
그것은 내부적으로 static_cast를 사용하는데, static_cast는 타입캐스트의 유효성을
동적으로 판단하지 않기 때문에, boost::polymorphic_downcast는 타입캐스트가
안전한 경우에만 사용되어야 한다.
디버그 빌드에서, boost::polymorphic_downcast는 실제로는, 타입 캐스트 유효성을
검증하기 위해 assert()함수를 사용하면서 dynamic_cast를 이용한다.
이러한 테스트는 NDEBUG 매크로가 정의된 디버그 빌드 경우에만 수행된다.

cross cast 를 위해서는 boost::polymorphic_cast 가 사용된다.
이것은 내부적으로 dynamic_cast 를 사용하는데, 그것이 cross cast를 수행하게 해주는
유일한 캐스트 연산자이기 때문이다.

boost::polymorphic_cast는 에러시 std::bad_cast타입의 예외를 던지기 때문에 선호된다.
반면, dynamic_cast의 경우는 캐스트 실패시 0을 리턴한다.
boost::polymorphic_cast를 사용하면, 수동으로 리턴값을 체크하지 않아도 된다.

boost::polymorphic_downcast 와 boost::polymorphic_cast 모두, 포인터를 캐스팅하는
경우에만 사용될수 있다. 이경우가 아니라면, dynamic_cast가 사용되어야 한다.

boost::polymorphic_downcast는 static_cast를 기반으로 구현되므로, 부모클래스의 객체들을
자식 클래스의 객체들로 변환시킬수는 없다. 또한, 포인터 타입이 아닌것을 캐스팅할때
boost::polymorphic_cast을 사용한다면, dynamic_cast에서 std::bad_cast를 발생시킨다.

모든 타입캐스트는 dynamic_cast를 이용해서 가능하기 때문에, boost::polymorphic_downcast 와
boost::polymorphic_cast를 꼭 사용해야만 하는것은 아니다. 하지만 Boost.Conversion 는
실제사용시 꽤 유용한 boost::lexical_cast 캐스트 연산자를 또한 제공한다.

#include <boost/lexical_cast.hpp>
#include <string>
#include <iostream>

int main()
{
   std::string s = boost::lexical_cast<std::string>(169);
   std::cout << s << std::endl;
   double d = boost::lexical_cast<double>(s);
   std::cout << d << std::endl;
}

boost::lexical_cast 를 사용하면, 여러가지 데이터 타입으로 캐스팅을 할수있다.
예제에서는 먼저 169를 스트링으로 캐스팅하고,
2번째 단계에서는 스트링을 부동 소수점 숫자로 변환시켰다.

boost::lexical_cast는 전환을 위해서, 내부적으로 스트림을 이용한다.
그러므로, 오버로드된 operator<<() 와 operator>>() 연산자를 가진 타입만이 전환 가능하다.
boost::lexical_cast를 사용하면 수동으로 스트림을 다룰 필요 없이,
타입변환이 코드 한줄 내에서 이루어진다. 또한 스트림을 사용하는것은 그 의도가 즉시
드러나지 않을 수 있도 있지만, boost::lexical_cast 같은 캐스트 연산자를 사용하면
코드의 의미를 좀더 잘 나타낼 수 있다.

만약 캐스팅에 실패하면, std::bad_cast를 상속받은 boost::bad_lexical_cast예외가 발생한다.

#include <boost/lexical_cast.hpp>
#include <string>
#include <iostream>

int main()
{
try
   {
     int i = boost::lexical_cast<int>("abc");
     std::cout << i << std::endl;
   }
   catch (boost::bad_lexical_cast &e)
   {
     std::cerr << e.what() << std::endl;
   }
}

"abc" 스트링이 int 타입의 숫자로 전환될수 없기 때문에 예외가 발생한다.


Boost.NumericConversion
숫자를 다른 타입의 숫자로 변환시 사용할수 있다.
C++에서 일반적으로 이러한 전환은 다음 예에서 보는 것처럼 암묵적으로 일어난다.

#include <iostream>

int main()
{
int i = 0x10000;
   short s = i;
   std::cout << s << std::endl;
}

위 예제는 int타입이 short타입으로 자동변환되고 에러없이 컴파일 된다.
비록 이 프로그램이 실행가능 하기 하지만, 그 결과는 해당 컴파일러와 그 구현에
의존적이기 때문에 예측할 수 없다.
i에 저장된 0x10000 이라는 숫자는 short로 전환되기엔 너무 크다.
C++표준에 따르면, 이 연산의 결과는 해당 구현에 따라 결정된다.
Visual C++ 2008에서 컴파일시에는 0을 리턴했다. 그리고 이값은 i의 값과는 분명 다르다.

이러한 전환시의 에러를 피하기 위해, boost::numeric_cast 을 사용할 수 있다.

#include <boost/numeric/conversion/cast.hpp>
#include <iostream>

int main()
{
   try
   {
     int i = 0x10000;
     short s = boost::numeric_cast<short>(i);
     std::cout << s << std::endl;
   }
   catch (boost::numeric::bad_numeric_cast &e)
   {
     std::cerr << e.what() << std::endl;
   }
}

출력:
bad numeric conversion: positive overflow

이것은 C++ 캐스트 연산자와 같은 방법으로 사용되었다.
그런데, 이것은 전환되는 숫자가 변형없이 캐스팅 가능한지 검증을 하게 된다.
예제에서 0x10000은 short 변수에 저장되기엔 너무 큰 수이기 때문에
boost::numeric::bad_numeric_cast 예외를 발생시킨다.
좀더 엄밀히 말하자면 boost::numeric::positive_overflow 타입의 예외가 발생된다.
이것은 양(+) 의수에 대해 오버플로우가 발생한 경우이다.
음(-)의 수에 대한 오버 플로우 발생시에는 boost::numeric::negative_overflow 예외가 발생된다.

#include <boost/numeric/conversion/cast.hpp>
#include <iostream>

int main()
{
try
   {
     int i = -0x10000;
     short s = boost::numeric_cast<short>(i);
     std::cout << s << std::endl;
   }
   catch (boost::numeric::negative_overflow &e)
   {
     std::cerr << e.what() << std::endl;
   }
}

Boost.NumericConversion에는 boost::numeric::bad_numeric_cast 를 상속받은
다른 예외들도 존재한다.
(boost::numeric::bad_numeric_cast은 std::bad_cast를 상속받았다)
