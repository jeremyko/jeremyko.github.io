boost::bind
기존 std::bind1st(), std::bind2nd() template 함수들을 단순화.
함수 시그니쳐(프로토타잎)이 틀린 경우 활용.
(STL 알고리즘에서 특정 함수 시그니쳐를 요구하는 경우에 활용가능)
그런데 함수 시그니쳐가 틀린데 호출되어야만 하는 경우란 언제인가?
예를 들어 다음과 같은 경우:


#include <iostream>
#include <vector>
#include <algorithm>

void print(int i)
{
   std::cout << i << std::endl;
}

int main()
{
   std::vector<int> v;
   v.push_back(1);
   v.push_back(3);
   v.push_back(2);

   std::for_each(v.begin(), v.end(), print);
}

STL알고리즘 std::for_each()는 3번째 인자로 인자 하나를 가지는 함수/함수자를 요구한다.
위의 예처럼 print함수는 인자 하나를 취하고 컨테이너내의 값들을 출력해줄것이다.
그런데 print함수대신 컨테이너의 모든 값들에대해 다음 처리를 해주고 싶다면?

void add(int i, int j)
{
std::cout << i + j << std::endl;
}

다른 signiture를 가진 함수를 넘겨야 하는 경우이다.이경우, 이항함수를 바로 전달할수 없다.
기존 STL에서는 기존 함수는 std::binary_function를 상속받는 함수객체로 수정되어야 한다.

#include <iostream>
#include <vector>
#include <algorithm>
#include <functional>

class add : public std::binary_function<int, int, void> //function object
{
public:
   void operator()(int i, int j) const
   {
        std::cout << i + j << std::endl;
   }
};

int main()
{
std::vector<int> v;
   v.push_back(1);
   v.push_back(3);
   v.push_back(2);
   std::for_each(v.begin(), v.end(), std::bind1st(add(), 10));
}

위와 같이 수정할수 있지만. 만약 boost:bind를 이용한다면 좀더 간단하다.

#include <boost/bind.hpp>
#include <iostream>
#include <vector>
#include <algorithm>

void add(int i, int j) //함수자일 필요없음.
{
std::cout << i + j << std::endl;
}

int main()
{
   std::vector<int> v;
   v.push_back(1);
   v.push_back(3);
   v.push_back(2);
   std::for_each(v.begin(), v.end(), boost::bind(add, 10, _1));
}

함수를 함수객체로 수정할 필요없이 bind의 첫번째 인자로 넣어주면 된다.
add함수가 2개의 인자를 필요로 하므로 bind에도 add함수에 들어갈 2개 인자를
지정해야 하는데, 첫번째는 10이고 나머지는 이상하게 보이는 _1 이다.
여기서 _1 은 placeholder라고 하고 _1은 bind가 단항함수로 변경됨을 의미한다.
(_2,_3 는 이항, 삼항 함수를 의미)

std::for_each() 실행될때, 단항함수(unary)signiture의 boost::bind가 호출되고,
실제 boost::bind 호출을 통해서는 별도의 함수, add()가 호출된다.
이때 add함수에 전달되는 인자는 첫번째 인자로 10, 두번째 인자로는
컨테이너v의 구성요소들의 값들이 대입될것이다.

다른 예로, 2개 인자를 가진 함수 시그니쳐를 요구하는 std::sort 경우에는
다음처럼 사용가능하다.

#include <boost/bind.hpp>
#include <vector>
#include <algorithm>

bool compare(int i, int j)
{
   return i > j;
}

int main()
{
   std::vector<int> v;
   v.push_back(1);
   v.push_back(3);
   v.push_back(2);
    std::sort(v.begin(), v.end(), boost::bind(compare, _1, _2));
}

이미 compare() 함수가 2개 인자를 가지고 있으므로 불필요해 보이지만,
compare함수 변경없이 정렬순서를 ascending으로 변경할때 bind가 유용하다.

...
int main()
{
   std::vector<int> v;
   v.push_back(1);
   v.push_back(3);
   v.push_back(2);
//placeholder 만 변경!
   std::sort(v.begin(), v.end(), boost::bind(compare, _2, _1));
}

만약 멤버함수를 바인딩 하고 싶다면, 모든 멤버함수(static 함수 제외)에게 암묵적으로 전달되는
this 인자를 명시적으로 넘겨줘야한다. 즉,
#include <iostream>
#include <boost/bind.hpp>
#include <vector>
#include <algorithm>

class someClass
{
public:
    someClass()
    {
        init();
    }

    void init()
    {
        v.push_back(1);
        v.push_back(3);
        v.push_back(2);
    }

    void callMemberFunc1()
    {
        std::for_each(v.begin(), v.end(), boost::bind(&someClass::memberFunc1, this,10, _1) ); // this
    }

    void memberFunc1(int i, int j)
    {
        std::cout <<"memberFunc1: "<< i + j << std::endl;
    }

    void memberFunc2(int i)
    {
        std::cout <<"memberFunc2: "<< i << std::endl;
    }

private:
    std::vector<int> v;
};

int main()
{
    someClass someClass;
    someClass.callMemberFunc1();

    //직접 멤버호출,
    boost::bind(&someClass::memberFunc2, someClass, _1) (99) ;
}

boost::ref
bind이용시 참조자로 전달되는 인자가 필요한경우 사용한다.
참조자는 복사불가이므로 필요하다.

#include <boost/bind.hpp>
#include <iostream>
#include <vector>
#include <algorithm>

void add(int i, int j, std::ostream &os)
{
os << i + j << std::endl;
}

int main()
{
   std::vector<int> v;
   v.push_back(1);
   v.push_back(3);
   v.push_back(2);

   std::for_each(v.begin(), v.end(), boost::bind(add, 10, _1, boost::ref(std::cout)));
}
