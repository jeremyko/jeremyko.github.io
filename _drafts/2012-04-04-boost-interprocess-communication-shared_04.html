boost::interprocess::managed_shared_memory

앞에서 설명된 boost::interprocess::shared_memory_object는
공유메모리에 바이트 단위로 읽고 쓰는 기능을 제공하지만, 별로 실용적이지 않다.
객체를 생성해서 저장하고 가져오는 방식을 사용하기 위해서
boost::interprocess::managed_shared_memory 가 제공된다.
이것을 이용하면 객체는 공유메모리상에 만들어지고, 다른 어플리케이션에서도
그 객체를 사용할수 있다.

#include <boost/interprocess/managed_shared_memory.hpp>
#include <iostream>

int main()
{
   boost::interprocess::shared_memory_object::remove("Highscore");
   boost::interprocess::managed_shared_memory
   managed_shm(boost::interprocess::open_or_create, "Highscore", 1024);
   int *i = managed_shm.construct<int>("Integer")(99);
   std::cout << *i << std::endl;
   std::pair<int*, std::size_t> p = managed_shm.find<int>("Integer");
   if (p.first)
     std::cout << *p.first << std::endl;
}

앞서의 사용 예에서는
데이터를 읽고 쓰기 위해, 공유메모리의 개별 바이트들로 직접 접근한 반면,
관리되는 공유메모리(managed shared memory)는 construct()같은 메서드를 사용한다.
이 메서드는 템플릿 인자로 데이터 타입을 필요로 한다.
그리고 생성되는 객체 구분을 위한 이름을 지정해야 한다.
예제에서는 "Integer" 를 지정했다.

construct()는 프록시 객체를 돌려주기 때문에, 생성된 객체를 초기화 시키기 위해서
입력인자를 전달할수 있다(즉, 99). 이것은 마치 생성자 호출처럼 보이기도 한다.
이를 통해서 객체의 생성 뿐 아니라 원하는값으로 초기화 가능하다.

managed shared memory 에서 특정 객체를 찾기 위해서 find() 를 이용한다.
객체의 이름을 인자로 넘기면, 객체에 대한 포인터를 돌려준다 실패시에는 0이 리턴된다.
예제에서 보다시피 find()호출로 돌려주는 실제 객체는 std::pair 타입이다.
찾고자 하는 객체에 대한 포인터는 first 속성으로 얻을수 있다.
그렇다면 second 속성으로 주어지는 것은 무엇인가?

#include <boost/interprocess/managed_shared_memory.hpp>
#include <iostream>

int main()
{
   boost::interprocess::shared_memory_object::remove("Highscore");
   boost::interprocess::managed_shared_memory managed_shm
   (boost::interprocess::open_or_create, "Highscore", 1024);
   int *i = managed_shm.construct<int>("Integer")[10](99);
   std::cout << *i << std::endl;
   std::pair<int*, std::size_t> p = managed_shm.find<int>("Integer");
   if (p.first)
   {
     std::cout << *p.first << std::endl;
     std::cout << p.second << std::endl;
   }
}

이번에는 int 타입 요소의 배열을 공유메모리에 생성하기 위해, construct() 호출후
[]에 둘러쌓인 10 을 제공했다. second 속성을 출력하면 10 이 출력된다.
(앞서의 예제에서는 second 속성은 1 이었다) 이렇게 생성된 배열은 모두 99로 초기화 된다.
개별적인 값으로 배열을 초기화 시키는 것은 불가능하다.
construct() 호출시에 이미 존재하는 경우에는 0이 리턴된다.
기존에 존재하는 managed shared memory에서 존재하는 객체를 재사용하려면
find_or_construct() 를 사용한다. 이경우엔 객체 초기화는 일어나지 않는다.

그빆에 construct()가 실패하는 경우가 있는데, 아래 예제에 나와있다.

#include <boost/interprocess/managed_shared_memory.hpp>
#include <iostream>

int main()
{
   try
   {
     boost::interprocess::shared_memory_object::remove("Highscore");
     boost::interprocess::managed_shared_memory
     managed_shm(boost::interprocess::open_or_create, "Highscore", 1024);
     int *i = managed_shm.construct<int>("Integer")[4096](99);
   }
   catch (boost::interprocess::bad_alloc &ex)
   {
     std::cerr << ex.what() << std::endl;
   }
}

위에서는 공유메모리에 1,024 바이트를 할당하고,  4,096 바이트의 int배열 객체를
생성하려 한다. 이경우엔 boost::interprocess::bad_alloc 예외가 발생한다.
공유메모리에 생성된 객체는 destroy() 를 이용해서 삭제한다.

#include <boost/interprocess/managed_shared_memory.hpp>
#include <iostream>

int main()
{
   boost::interprocess::shared_memory_object::remove("Highscore");
   boost::interprocess::managed_shared_memory
   managed_shm(boost::interprocess::open_or_create, "Highscore", 1024);
   int *i = managed_shm.find_or_construct<int>("Integer")(99);
   std::cout << *i << std::endl;
   managed_shm.destroy<int>("Integer");
   std::pair<int*, std::size_t> p = managed_shm.find<int>("Integer");
   std::cout << p.first << std::endl;
}

destroy() 메서드 인자로 객체 이름을 전달한다.
객체 삭제 성공시 bool 타입 true를 리턴할것이다. 객체가 존재하면 항상 삭제할것이므로,
만약 객체가 존재하지 않는 경우엔 false가 리턴된다.

destroy_ptr() 메서드는객체 삭제를 위해서 객체 포인터를 인자로 넘긴다.
배열의 삭제에도 사용된다.

managed shared memory 는 서로다른 어플리케이션끼리 공유된 객체의 저장을 쉽게 해주기
때문에, C++ 표준 템플릿 라이브러리(STL)의 컨테이너 사용도 자연스러운 일이다.
그런데 이러한 컨테이너들은 각 객체의 할당을 위해 개별적으로 new 를 호출한다.
이 컨테이너들을 managed shared memory 내에서 사용하기 위해서는
공유 메모리내에서 메모리를 할당하게끔 알려줘야 한다.

불행하게도, 많은 STL 의 구현들이, 예를들면 std::string 혹은 std::list 같은,
Boost.Interprocess 와 같이 사용되기에는 부적절하다.
컨테이너를 사용을 위해 좀더 유연한 구현들이 boost::interprocess 네임스페이스에
존재한다. 예를 들어 boost::interprocess::string 는 std::string 와 동일하게 작동하면서
객체가 managed shared memory 안에 안전하게 저장할수 있게 해준다.

#include <boost/interprocess/managed_shared_memory.hpp>
#include <boost/interprocess/allocators/allocator.hpp>
#include <boost/interprocess/containers/string.hpp>
#include <iostream>

int main()
{
   boost::interprocess::shared_memory_object::remove("Highscore");
   boost::interprocess::managed_shared_memory
   managed_shm(boost::interprocess::open_or_create, "Highscore", 1024);

   typedef boost::interprocess::allocator<char,
   boost::interprocess::managed_shared_memory::segment_manager> CharAllocator;

   typedef boost::interprocess::basic_string<char, std::char_traits<char>, CharAllocator> string;

   string *s = managed_shm.find_or_construct<string>("String")
   ("Hello!", managed_shm.get_segment_manager());
   s->insert(5, ", world");
   std::cout << *s << std::endl;
}

string을 생성시 동일한 managed shared memory에 메모리가 할당되게 하려면
C++ standard 기본 할당자가 아닌 별도의 할당자를 제공해야한다.
boost::interprocess::allocator 를 이용해서 할당자를 생성할수 있는데
이렇게 만들어진 할당자는 내부적으로 managed shared memory의
segment manager 을 사용하는데, managed shared memory의 메모리를 관리를 책임진다.

새로운 할당자를 이용하기 위해서, 예제처럼 std::basic_string 가 아닌
boost::interprocess::basic_string 가 사용된다.
그리고 find_or_construct() 인자로 segment manager의 포인터를 전달해야한다.
string 뿐 아니라 그밖에도 잘알려진 다른 컨테이너를 위한 구현도 제공되는데,
예를 들면 boost::interprocess::vector, boost::interprocess::map 등이 있다.
(boost/interprocess/containers/vector.hpp, boost/interprocess/containers/map.hpp 에 정의됨)

각각의 어플리케이션에서 동일한 공유메모리에 접근할때마다, 생성, 검색, 삭제와 같은
동작들은 자동적으로 동기화 된다.
만약 2개의 어플리케이션들이 공유메모리에 서로다른 이름의 객체 생성을 시도한다면
그 오퍼레이션은 적절한 순서로 시리얼하게 발생한다.
만약 하나의 어플리케이션에서 여러개의 동작을 다른 어플리케이션에 의한 동작에 방해받지 않고
동시에 실행되기를 원한다면 atomic_func() 를 사용할수 있다.

#include <boost/interprocess/managed_shared_memory.hpp>
#include <boost/bind.hpp>
#include <iostream>

void construct_objects(boost::interprocess::managed_shared_memory &managed_shm)
{
   managed_shm.construct<int>("Integer")(99);
   managed_shm.construct<float>("Float")(3.14);
}

int main()
{
   boost::interprocess::shared_memory_object::remove("Highscore");
   boost::interprocess::managed_shared_memory
   managed_shm(boost::interprocess::open_or_create, "Highscore", 1024);

   managed_shm.atomic_func(boost::bind(construct_objects, boost::ref(managed_shm)));

   std::cout << *managed_shm.find<int>("Integer").first << std::endl;
   std::cout << *managed_shm.find<float>("Float").first << std::endl;
}

atomic_func()은 입력인자과 리턴값이 없는 함수를 필요로 한다.
전달된 함수는 독점적인 공유 메모리 접근이 가능하지만, 모든 동작이 독점적인것은 아니고
객체 생성, 검색, 삭제 동작에 대해서만 가능하다. 만약 다른 어플리케이션이 공유메모리상의
객체에 대한 포인터를 가지고 있다면, 이 객체에 접근해서 변경할수 있다.

객체 접근에 대한 동기화처리에도 Boost.Interprocess 가 사용될수 있는데 다음장에서 살펴본다.
