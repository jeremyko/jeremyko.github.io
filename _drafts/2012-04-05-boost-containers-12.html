Boost.Array

배열을 위한 템플릿 클래스, 엘리먼트 수가 고정된 std::vector 라고 생각해도 무방.

#include <boost/array.hpp>
#include <iostream>
#include <string>
#include <algorithm>

int main()
{
  typedef boost::array<std::string, 3> array;
   array a;
   a[0] = "Boris";
   a.at(1) = "Anton";
   *a.rbegin() = "Caesar";
   std::sort(a.begin(), a.end());
   for (array::const_iterator it = a.begin(); it != a.end(); ++it)
        std::cout << *it << std::endl;

   std::cout << a.size() << std::endl;
    std::cout << a.max_size() << std::endl;
}

예제에서 보다시피 사용법이 매우 명확해서 별다른 설명이 필요없지만, 다음예제에서는 약간 특이한점을 발견할수 있다.

#include <boost/array.hpp>
#include <string>

int main()
{
  typedef boost::array<std::string, 3> array;
  array a = { "Boris", "Anton", "Caesar" };
}

즉, 전통적인 c++ 처럼 초기화될수 있다. 이 컨테이너는 C++표준 Technical Report 1에 추가되었기 때문에 std::array 로 사용할수 있다 (물론..TR1을 구현한 컴파일러에 한함).

Boost.Unordered

Boost.Unordered 는 표준 라이브러리 std::set, std::multiset, std::map 그리고 std::multimap 을 보완하는 추가적인 4가지 클래스,즉, boost::unordered_set, boost::unordered_multiset, boost::unordered_map 그리고 boost::unordered_multimap 을 제공한다. 이것들은 표준라이브러리것과 비교해서 큰 차이점은 없고, 비슷한 인터페이스를 가진다. 차이점은 boost컨테이너들은 정렬을 수행하지 않고, 따라서 앨리먼트는 정렬가능할 필요가 없다는 점이다.

Boost.Unordered 는 내부 앨리먼트들의 정렬이 중요하지 않은 경우에 유용하다. 컨테이너 내부에서 앨리먼트를 빠르게 찾기 위해 hash값이 계산되어진다. 이 고유한 숫자값을 계산하기 위해, Boost.Unordered 컨테이너에 저장될 데이터 타입은 이러한 계산을 지원해야 한다. 즉, std::set 은 앨리먼트가 정렬가능할것을 요구하는데 비해, boost::unordered_set 는 앨리먼트가 hash값을 지원하기를 요구한다. 이같은 요구조건에도 불구하고, 정렬이 요구되지 않는다면  Boost.Unordered가 선호된다.

다음은 boost::unordered_set 의 예제이다.

#include <boost/unordered_set.hpp>
#include <iostream>
#include <string>

int main()
{
  typedef boost::unordered_set<std::string> unordered_set;
    unordered_set set;

   set.insert("Boris");
   set.insert("Anton");
   set.insert("Caesar");

   for (unordered_set::iterator it = set.begin(); it != set.end(); ++it)
        std::cout << *it << std::endl;

   std::cout << set.size() << std::endl;
   std::cout << set.max_size() << std::endl;

   std::cout << (set.find("David") != set.end()) << std::endl;
   std::cout << set.count("Boris") << std::endl;
}

출력:
Caesar
Anton
Boris
3
805306456
0
1

보다시피 std::set 과 비슷한 메서드를 제공한다. 예제는 큰 변경없이 std::set 로 대체될수 있다.

다음은 boost::unordered_map 를 이용해서 개인의 이름과 나이를 저장하는 예이다.

#include <boost/unordered_map.hpp>
#include <iostream>
#include <string>

int main()
{
    typedef boost::unordered_map<std::string, int> unordered_map;
    unordered_map map;

    map.insert(unordered_map::value_type("Boris", 31));
   map.insert(unordered_map::value_type("Anton", 35));
   map.insert(unordered_map::value_type("Caesar", 25));

   for (unordered_map::iterator it = map.begin(); it != map.end(); ++it)
        std::cout << it->first << ", " << it->second << std::endl;

   std::cout << map.size() << std::endl;
    std::cout << map.max_size() << std::endl;
   std::cout << (map.find("David") != map.end()) << std::endl;
   std::cout << map.count("Boris") << std::endl;
}

출력:
Caesar, 25
Anton, 35
Boris, 31
3
805306456
0
1

역시 std::map 과 비교해서 큰 차이가 없다. std::map 로 바로 교체 가능하다.

앞서 설명한것처럼 컨테이너에 저장되는 데이터는 hash값을 지원해야 한다고 했다. std::string 등은 원래 이 값을 지원하기 때문에 가능했다. 만약 사용자 정의 데이터를 사용하고자 한다면 hash함수를 수동으로 정의 해야한다. 즉,

#include <boost/unordered_set.hpp>
#include <string>

struct person
{
   std::string name;
    int age;

   person(const std::string &n, int a)   : name(n), age(a)
   {
   }
   bool operator==(const person &p) const
   {
        return name == p.name && age == p.age;
   }
};

std::size_t hash_value(person const &p)
{
   std::size_t seed = 0;
    boost::hash_combine(seed, p.name);
   boost::hash_combine(seed, p.age);
   return seed;
}

int main()
{
    typedef boost::unordered_set<person> unordered_set;
    unordered_set set;

   set.insert(person("Boris", 31));
    set.insert(person("Anton", 35));
   set.insert(person("Caesar", 25));
}

해시함수 이름은 hash_value() 로 정의된다. 하나의 입력인자를 통해, 해쉬값을 정의할 데이터 타입의 객체를 받는다.
객체에 대해 hash값 계산이 필요할때마다 이 함수가 자동으로 호출된다. 

Boost C++ Libraries 는 std::string과 같은 다양한 데이터 유형에 대한 함수를 이미 정의해 놓았다. 하지만 person같은 사용자 정의 데이터인 경우는 수동으로 지정해야 한다. hash_value() 함수의 구현은 간단하다. 해쉬값은 객체 프로퍼티에 하나씩 boost::hash_combine() 을 적용해서 구해진다. 해시 라이브러리는 boost/functional/hash.hpp 에 정의되있고, Boost.Unordered 를 사용하면 자동으로 포함될것이다 (모든 컨테이너가 해쉬값 계산을 위해 Boost.Hash를 필요로 하기 때문). 사용자 데이터인 경우, hash_value() 구현과 더불어 == operator도 구현이 필요하다.
