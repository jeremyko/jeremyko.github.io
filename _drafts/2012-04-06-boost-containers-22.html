Boost.MultiIndex

이것은 지금까지 설명된 컨테이너보다 좀 복잡하다.
Boost.Array나 Boost.Unordered는 즉시 사용가능한 컨테이너를 제공하는 반면,
Boost.MultiIndex는 새로운 컨테이너를 정의할수 있게 해준다.

C++표준 컨테이너와 반대로, 사용자 정의 컨테이너는 
데이터에 대해 임의갯수의 인터페이스를 제공할수 있다.
예를 들어 사용자는 std::map과 흡사한 컨테이너를 만들수 있는데, 
key에 값을 할당할수 있을뿐 아니라 또한 반대로 값을 key처럼 이용할수도 있다. 
Boost.MultiIndex 도움없이 이같은 기능을 수행하려면
2개의 std::map이 필요할것이고 두 컨테이너의 데이터 일치를 위한 
동기화 작업이 필요할것이다.

다음예는 Boost.MultiIndex를 이용해서 사람의 나이와 이름을 저장하는 
새로운 컨테이너를 생성한다.
std::map와는 다르게 이름과 나이 모두를 이용해서 검색이 가능하다.

#include <boost/multi_index_container.hpp>
#include <boost/multi_index/hashed_index.hpp>
#include <boost/multi_index/member.hpp>
#include <iostream>
#include <string>

struct person
{
    std::string name;
    int age;

    person(const std::string &n, int a) : name(n), age(a)
    {
    }
};

typedef boost::multi_index::multi_index_container
    <
        person,
        boost::multi_index::indexed_by
        <
            boost::multi_index::hashed_non_unique
            < 
                boost::multi_index::member<person, std::string, &person::name>   
            >,
            boost::multi_index::hashed_non_unique
            <
                boost::multi_index::member<person, int, &person::age>
            >
        >
    > person_multi;

int main()
{
    person_multi persons;

    //암묵적 인터페이스 지정
    persons.insert(person("Boris", 31));
    persons.insert(person("Anton", 35));
    persons.insert(person("Caesar", 25));

    std::cout <<"count Boris:"<< persons.count("Boris") << std::endl;

    //명시적 인터페이스 지정, 인터페이스 번호는 1
    const person_multi::nth_index<1>::type &age_index = persons.get<1>();
    std::cout << "count age 25:" << age_index.count(25) << std::endl;
}

출력:
count Boris:1
count age 25:1

앞서 언급한대로 Boost.MultiIndex는 특정 컨테이너를 제공하는것이 아니라
새로운 컨테이너를 만들기 위한 클래스를 제공한다.
일반적으로 첫번째 단계는, typedef와 Boost.MultiIndex의 여러 클래스를 이용해서,
새로운 컨테이너를 정의하는 것이다.

이때 boost::multi_index::multi_index_container 가 사용된다.
이것은 템플릿 클래스이기 때문에 최소 2개의 인자가 필요하다.
첫번째 인자는 컨테이너가 담을 데이터 유형이다
(위 예제에서는 person이라는 사용자정의 클래스가 사용됨).
두번째 인자는 컨테이너가 제공해야하는 여러개의 서로다른 인덱스를 나타내기 위해 사용된다.

Boost.MultiIndex에 의한 컨테이너의 장점은, 서로다른 인터페이스를 통해서
데이터에 접근할수 있다는것이다.
해당 컨테이너의 인터페이스 갯수와 유형은 이제 사용자가 정의할수 있다.

예제의 컨테이너는 나이 혹은 이름으로 찾기 위해 2개의 인터페이스를 정의 했다.
Boost.MultiIndex는 이 인터페이스들의 인덱스를 호출한다.

인터페이스들은 boost::multi_index::indexed_by 템플릿 클래스에 의해 정의된다.
각각의 인터페이스가 이 클래스의 인자로 전달된다.
예제에서는 boost::multi_index::hashed_non_unique 타입의 2개의 인터페이스들을 정의했다.

만약 컨테이너가 Boost.Unordered 처럼(즉,내부적으로 앨리먼트 저장시 해쉬값을 이용)
동작해야 한다면, 이 인터페이스가 사용된다.

boost::multi_index::hashed_non_unique 템플릿 클래스도 역시 하나의 인자가 필요한데,
해쉬값을 계산할수 있는 클래스를 요구한다.

2개의 인터페이스들이 이름과 나이로 데이터 접근을 허용해야 하기 때문에 인터페이스 하나는
이름으로 해쉬값을 계산하고, 나머지 인터페이스는 나이로 해쉬값을 각각 계산한다.

저장되는 데이터의 속성에 접근하기 위해 
핼퍼 템플릿 클래스 boost::multi_index::member 가 제공된다.
예제에서, boost::multi_index::member 에게 person클래스의 어떤 속성에 접근해야하고
그 속성의 타입이 어떤것인지 알려주기 위해 여러개의 입력인자가 사용되었다.

person_multi 정의가 처음에는 꽤 복잡해보이지만, 그 클래스 자체는 boost::unordered_map
과 비슷하게 동작한다. 결정적인 차이는 이름과 나이로 모두 검색이 가능하다는 점이다.

MultiIndex 컨테이너에 접근하기 위해서는 인터페이스를 지정해야한다.
(단 하나의 예외가 있는데, 첫번째 정의된 인터페이스가 그것이다. 만약 person객체가
insert() 나 count()로 직접 접근된다면 첫번째 인터페이스가 암묵적으로 사용된다.)
예제에서 name 속성이 첫번째 인터페이스이므로 명시되지 않았다.
하지만 두번째로 정의된 age 인터페이스를 이용해서 컨테이너에 접근하려면 명시적인 지정이 필요하다.
이때는 인터페이스 번호가 필요하다.
이 인덱스는 0부터 시작되고, boost::multi_index::indexed_by 에 전달된 순서대로 부여된다.
예제에서처럼 두번째 인터페이스에 접근하기 위해서는 get()메서드를 호출하면서
템플릿인자로 사용할 인터페이스 번호를 넘기면 된다.

get()이 리턴하는 값은 좀 복잡해보인다. 그것은 nth_index 클래스를 리턴하는데,
이 역시 템플릿 클래스로, MultiIndex 컨테이너에 접근하기 위해서는
get()에 전달한 동일한 인터페이스 번호를 입력인자로 넘겨야한다.
그리고 nth_index::type을 통해서 해당되는 인터페이스 유형을 얻을수 있다.
예제에서 age_index 는 사람을 나이로 찾기 위한 해쉬 인터페이스가 된다.

한편, 이름과 나이 정보가 인터페이스에 의해 MultiIndex 컨테이너의 키값들이 될수있으므로,
그것들은 더이상 임의로 변경될수 없다. 만약 이름으로 검색된후 나이가 변경된다면
나이를 이용하는 인터페이스는 키값이 변경되었음을 알지 못할 것이다.
이경우 새로운 해쉬값 계산이 필요하다.

std::map에서 키값이 변경 불가인것처럼,MultiIndex 컨테이너에 저장된 정보들도 변경할수 없다.
엄밀히 말하자면 MultiIndex 컨테이너에 에 저장된 데이터는 상수이다.

삭제하고 수정된 데이터를 저장하는 수고를 덜기 위해, 직접 변경을 위한 유용한 메서드가 제공된다.

#include <boost/multi_index_container.hpp>
#include <boost/multi_index/hashed_index.hpp>
#include <boost/multi_index/member.hpp>
#include <iostream>
#include <string>

struct person
{
    std::string name;
    int age;

    person(const std::string &n, int a) : name(n), age(a)
    {
    }
};

typedef boost::multi_index::multi_index_container
    <
        person,
        boost::multi_index::indexed_by
        <
            boost::multi_index::hashed_non_unique
            <
                boost::multi_index::member<person, std::string, &person::name>
            >,
            boost::multi_index::hashed_non_unique
            <
                boost::multi_index::member<person, int, &person::age>
            >
        >
    > person_multi;

void set_age(person &p)
{
    p.age = 32;
}

int main()
{
    person_multi persons;

    persons.insert(person("Boris", 31));
    persons.insert(person("Anton", 35));
    persons.insert(person("Caesar", 25));

    person_multi::iterator it = persons.find("Boris");
    persons.modify(it, set_age);

    const person_multi::nth_index<1>::type &age_index = persons.get<1>();
    std::cout << "count age32:"<< age_index.count(32) << std::endl;
}

출력:
count age32:1

모든 Boost.MultiIndex 인터페이스는 컨테이너에 직접동작하는 modify() 메서드를 지원한다.
변경할 객체는 iterator 를 이용해서 첫번째 인자로 전달된다.
두번째 인자는 첫번째 인자로 넘긴 유형의 객체를 인자로 하는 함수나 함수객체이다.
예제에서 set_age() 함수가 객체를 변경한다.

지금까지는 해쉬값 계산을 위한 하나의 인터페이스만을 소개했다
(boost::multi_index::hashed_non_unique).

데이터가 2번 저장되는것을 방지하기 위해서
boost::multi_index::hashed_unique 를 대신 사용할수 있다.
알아둘점은 모든 인터페이스 조전을 만족시키지 않으면 저장이 불가하다는 점이다.

#include <boost/multi_index_container.hpp>
#include <boost/multi_index/hashed_index.hpp>
#include <boost/multi_index/member.hpp>
#include <iostream>
#include <string>

struct person
{
    std::string name;
    int age;

    person(const std::string &n, int a) : name(n), age(a)
    {
    }
};

typedef boost::multi_index::multi_index_container
    <
        person,
        boost::multi_index::indexed_by
        <
            boost::multi_index::hashed_non_unique
            <
                boost::multi_index::member<person, std::string, &person::name>
            >,
            boost::multi_index::hashed_unique
            <
                boost::multi_index::member<person, int, &person::age>
            >
        >
    > person_multi;

int main()
{
    person_multi persons;

    persons.insert(person("Boris", 31)); //OK
    persons.insert(person("Anton", 31)); //SKIP
    persons.insert(person("Caesar", 25));

    const person_multi::nth_index<1>::type &age_index = persons.get<1>();
    std::cout << "count age31:"<< age_index.count(31) << std::endl;
}

출력:
count age31:1

이제 컨테이너는 boost::multi_index::hashed_unique 클래스를 두번째 인터페이스로
사용한다. 이제 두 사람은 동일한 나이를 가질수 없게 되었다.
예제에서 Anton이라는 인물을 동일한 나이로 저장하려 하는데, 고유한 해쉬값을 가져야 하는
두번째 인터페이스를 지키지 않았기 때문에, 데이터가 저장되지 않는다.
그러므로 나이 31인 사람을 검색하면 1을 리턴할것이다.

다음 예제는 Boost.MultiIndex가 제공하는 나머지 3가지 인터페이스를 소개한다.
boost::multi_index::sequenced
boost::multi_index::ordered_non_unique
boost::multi_index::random_access

#include <boost/multi_index_container.hpp>
#include <boost/multi_index/sequenced_index.hpp>
#include <boost/multi_index/ordered_index.hpp>
#include <boost/multi_index/random_access_index.hpp>
#include <boost/multi_index/member.hpp>
#include <iostream>
#include <string>

struct person
{
    std::string name;
    int age;

    person(const std::string &n, int a) : name(n), age(a)
    {
    }
};

typedef boost::multi_index::multi_index_container
<
   person,
   boost::multi_index::indexed_by
   <
     boost::multi_index::sequenced<>,
     boost::multi_index::ordered_non_unique
     <
       boost::multi_index::member<person, int, &person::age>
     >,
     boost::multi_index::random_access<>
   >
> person_multi;

int main()
{
    person_multi persons;

    persons.push_back(person("Boris", 31));
    persons.push_back(person("Anton", 31));
    persons.push_back(person("Caesar", 25));

    const person_multi::nth_index<1>::type &ordered_index = persons.get<1>();
    person_multi::nth_index<1>::type::iterator lower = ordered_index.lower_bound(30);
    person_multi::nth_index<1>::type::iterator upper = ordered_index.upper_bound(40);
    for (; lower != upper; ++lower)
        std::cout << "lower->name:"<< lower->name << std::endl;

    const person_multi::nth_index<2>::type &random_access_index = persons.get<2>();
    std::cout << "random_access_index 2:" << random_access_index[2].name << std::endl;
}

출력:
lower->name:Boris
lower->name:Anton
random_access_index 2:Caesar

boost::multi_index::sequenced 를 이용하면 MultiIndex 컨테이너를 std::list 처럼 사용할수 있다.
이 인터페이스는 꽤 사용하기 간단해서, 템플릿 인자 전달 필요가 없다.
person 객체는 순서대로 저장 된다.

boost::multi_index::ordered_non_unique 인터페이스를 사용하면, 객체들은 자동적으로 정렬된다.
이 인터페이스에는 정렬을 위한 기준을 제공해줘야 하는데, 예제에서는
boost::multi_index::member 헬퍼 클래스를 사용해서 나이로 정렬하게 한다.
lower_bound() 과 upper_bound()를 사용하면 저장된 정보들을 특정 범위에서 검색할수 있다.
이러한 메서드는 정렬이 먼저 되야하므로, 다른 인터페이스에서는 제공되지 않는다.

boost::multi_index::random_access를 이용하면 MultiIndex 컨테이너를 std::vector 처럼 사용할수 있다.
중요한 메서드는 operator[]() 와 at() 이다.
알아둘점은 이 인터페이스는 boost::multi_index::sequenced 인터페이스를 모두 포함하고 있다는 것이다.
boost::multi_index::random_access 를 사용하면서 boost::multi_index::sequenced의
모든 메서드를 사용할수 있다.

key extractors 

이제 key extractors (key추출자) 에 대해 알아본다.
이것은 객체에 대한 참조를 받아서 그것과 연관된 키값을 돌려주는 function object이다.
이미 우리가 알고 있는것이 하나 있는데, boost::multi_index::member 이다.
이 헬퍼 클래스가 key추출자 라고 불리는데, 왜냐하면 클래스에서 어떤 속성이
인터페이스의 키로 쓰여야 하는지 지정할수 있게  해주기 때문이다.

다음예제에서 2가지 새로운 key extractor를 볼수 있다.

#include <boost/multi_index_container.hpp>
#include <boost/multi_index/ordered_index.hpp>
#include <boost/multi_index/hashed_index.hpp>
#include <boost/multi_index/identity.hpp>
#include <boost/multi_index/mem_fun.hpp>
#include <iostream>
#include <string>

class person
{
public:
   person(const std::string &n, int a) : name(n), age(a)
   {
   }

   bool operator<(const person &p) const
   {
     return age < p.age;
   }

   std::string get_name() const
   {
     return name;
   }

private:
   std::string name;
   int age;
};

typedef boost::multi_index::multi_index_container
<
   person,
   boost::multi_index::indexed_by
   <
     boost::multi_index::ordered_unique
     <
     boost::multi_index::identity<person>
     >,
     boost::multi_index::hashed_unique
     <
       boost::multi_index::const_mem_fun<person, std::string, &person::get_name>
           >
   >
> person_multi;

int main()
{
   person_multi persons;

   persons.insert(person("Boris", 31));
   persons.insert(person("Anton", 31));
   persons.insert(person("Caesar", 25));

   std::cout <<"ordered begin:"<< persons.begin()->get_name() << std::endl;

   const person_multi::nth_index<1>::type &hashed_index = persons.get<1>();
   std::cout <<"hashed_index count \"Boris\":"<<hashed_index.count("Boris") << std::endl;
}

출력:
ordered begin:Caesar
hashed_index count "Boris":1

key추출자 boost::multi_index::identity 는 컨테이너에 저장된 데이터 타입들을
키로 이용하기 위해 사용된다. 이것은 person 클래스가 정렬가능해야 함을 요구하는데,
boost::multi_index::ordered_unique 인터페이스를 위한 키로 지정되었기 때문이다.
예제에서는 person클래스에서 operator<() 연산자를 오버로딩하고 있다.

boost::multi_index::const_mem_fun 과 boost::multi_index::mem_fun key추출자를 이용하면
메서드의 리턴값을 키값으로 이용할수 있다. 두 개의 차이는 const여부이다.

그밖에 Boost.MultiIndex에서는 2개의 key추출자를 더 제공한다.
boost::multi_index::global_fun : 비 클래스 함수(global function)나 정적 메서드를 위함.
boost::multi_index::composite_key : 여러 key추출자로 구성된 key추출자 생성.

http://www.boost.org/doc/libs/1_49_0/libs/multi_index/doc/tutorial/key_extraction.html

// 별로 실용적이지 않은, 급조된 boost::multi_index::global_fun 예제...
#include <boost/multi_index_container.hpp>
#include <boost/multi_index/ordered_index.hpp>
#include <boost/multi_index/global_fun.hpp>

struct someStruct
{
    int nId;
    std::string val;
};

int judge(const someStruct& r)
{   //음수로 만들어 정렬순서를 반대로 만듬..
   return  r.nId * (-1);
}

typedef boost::multi_index::multi_index_container
<
   someStruct,
   boost::multi_index::indexed_by
   < // sort by nVal
     boost::multi_index::ordered_non_unique
     <
     boost::multi_index::global_fun<const someStruct&, int,&judge>
     > ,
            boost::multi_index::hashed_unique
            <
                boost::multi_index::member<someStruct, std::string, &someStruct::val>
            >
   >
> someStruct_container;

int main()
{
   someStruct_container rectContainer;

    someStruct rectA,rectB,rectC;
    rectA.nId = 1;
    rectA.val = "A";
    rectB.nId = 2;
    rectB.val = "BB";
    rectC.nId = 3;
    rectC.val = "CCC";

   rectContainer.insert(rectA);
   rectContainer.insert(rectB);
   rectContainer.insert(rectC);

    std::cout <<"ordered begin:"<< rectContainer.begin()->nId << std::endl;
    someStruct_container::iterator it = rectContainer.find(-2);
    std::cout <<"id:"<< it->nId <<" val:"<< it->val << std::endl;

    someStruct_container::nth_index<1>::type::iterator it1=  rectContainer.get<1>().find("CCC");
    //typedef someStruct_container::nth_index<1>::type val_index;
    //val_index::iterator it=  rectContainer.get<1>().find("CCC");
    std::cout <<"id:"<< it1->nId <<" val:"<< it1->val << std::endl;
}

출력:
ordered begin:3
id:2 val:BB
id:3 val:CCC
