virtualbox에서 guest os 로 ubuntu server 16.04 버전을 설치하고 os x 호스트에서 ssh 를 이용해서 접속하려고 시도했는데, 이상하게도 윈도우 호스트에서는 잘되던것이 mac 에서는 잘안되서... 여러번 삽질하던 끝에 겨우 성공했다.
이건 뭐 개발도 아니고 그냥 tip 인데 혹시 나처럼 또 삽질하는 사람 없기를 바라면서 올려본다.

일단 내 환경은 다음과 같다

macOS Sierra 10.12.1
VirtualBox 5.0.30 r112061

먼저 guest os (ubuntu server) 에서 network interface를 확인해본다.

eth0, eth1 이런게 안나오고 enp0s8, lo 이거 2개만 나온다. Predictable Network Interface Names 이라고 하는데 좀 생소하다. 만약 eth0, eth1같은 기존 명칭을 사용하고 싶다면 다음 파일을 아래처럼 편집하면 된다
    $vi /etc/default/grub
    GRUB_CMDLINE_LINUX_DEFAULT="net.ifnames=0 biosdevname=0"
    $sudo update-grub
    $sudo reboot

여기서는 기존대로사용하는것으로한다.

/etc/network/interfaces 를 열어서 다음처럼 enp0s8 을 추가한다(enp0s8 이란것이 eth1 이다)

auto enp0s8
iface enp0s8 inet static
address 192.168.56.2
netmask 255.255.255.0
그리고 guest os를 shutdown 한다

virtualbox 에서 설정항목에서 네트워크항목, 어댑터 1 (NAT)에서 고급 -> 포트 포워딩 설정에서 다음처럼 포트를 설정해준다


다시, 어댑터 2 를 사용함으로 체크하고 호스트 전용 어댑터로 설정해준다.


이제 ubuntu server를 재 시작하고 mac 터미널에서 다음처럼 접속한다.

ssh -p 2222 root@192.168.56.1
