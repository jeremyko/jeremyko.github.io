boost::format

std::printf()에 type safe과 확장성을 개선. std::printf()는 다음과 같이  type에 안전하지 않다.


#include <iostream>
#include <boost/format.hpp>

int main()
{
    std::printf("x,y=(%d,%d) \n",-1,2);    
    std::cout << boost::format("x,y=(%d,%d)") % -1 % 2 << std::endl; //기존 print format
    std::cout << boost::format("x,y=%||,%||") % -1 % 2 << std::endl;
    std::cout << boost::format("x,y=(%1%,%2%)") % -1 % 2 << std::endl;//placeholder를 지정

std::printf("x,y=(%d,%d) \n",-1.5, 2.5); // Incorrect!
    std::cout << boost::format("x,y=(%1%,%2%)") % -1.5 % 2.5  << std::endl;  
}

출력:
x,y=(-1,2)
x,y=(-1,2)
x,y=(0,-1074266112)
x,y=(-1.5,2.5)

문법은 다음과 같다.
boost::format( format-string ) % arg1 % arg2 % ... % argN

format-string은 3가지 형태로 사용된다.

1. legacy print format : 몇가지 호환 안되는 점이 있음.
    
2. %N% : %1%, %2%처럼 placeholder를 지정

3. %|spec| => type-conversion character를 처리하기 위해서 필요.
   spec은 다음 형식을 따른다.
  [ N$ ] [ flags ] [ width ] [ . precision ] type-char
   N$만 설명하면, positional format specification라고 불린다.
   N은 적용될 아규먼트를 나타낸다. 즉 1$면 첫번째 인자에 적용된다는 의미.
   
첫번째 사용은 앞서의 예제에 있는것을 참조하면 알수 있을것이다.
두번째 placeholder를 지정하는 방식은 format생성자로 %1,%2등과 같은 placeholder가
지정된 문자열을 넘기고 실제 데이터는 % 연산자를 사용해서 전달된다.
앞의 예제에서 만약 placeholder를 다음과 같이 변경하면,
...
std::cout << boost::format("x,y=(%2%,%1%)") % -1 % 2  << std::endl;
...

출력은 다음과 같다.
x,y=(2,-1)

c++ manipulator 를 이용해서 데이터를 포맷할경우 boost::io::group()를 사용한다. 

// 예) 출력되는 양수값에 +부호를 붙이는 경우
#include <boost/format.hpp>
#include <iostream>

int main()
{
   std::cout << boost::format("%1% %2% %1%") % boost::io::group(std::showpos, 99) % 100 << std::endl;
}  
출력:
+99 100 +99

그런데 +가 첫번째만 표시되게 하고 싶다면, 앞에서 기술된 %|spec| 형태로 format생성자를 호출하면 된다.

int main()
{  
   std::cout << boost::format("%|1$+| %2% %1%") % 99 % 100 << std::endl;   
  
   //혹은
   std::cout << boost::format("%|+| %|| %||") % 99 % 100 % 99 << std::endl;
   // %|spec| => spec은 일부/전체 생략 가능하다.
}

%1% 가 %|1$+| 로 교체되었다.
2개의 pipe표시가 추가됨과 아울러, 데이터에 대한 참조(1% 가 아닌 1$로 표현된)
또한 pipe 안에 표시되었다.

즉, 앞서의 예에서
std::cout << boost::format("x,y=(%1%,%2%)") % -1 % 2  << std::endl;

은 다음처럼 쓸수도 있다.
std::cout << boost::format("x,y=%||,%||") % 99 % 100 << std::endl;

1$과 같은 데이터에 대한 참조는 선택적이지만, 지켜야할것은 placeholder에 대해
모두 제공하거나 아니면 모두 제공하지 않는다는 것이다(일부만 사용할수 없다).
즉, 다음처럼 2,3번째 placeholder에 대해서만 데이터 참조들이 제공되고 첫번째에 대해서는
생략되었다면 실행시 오류가 발생한다.

#include <boost/format.hpp>
#include <iostream>

int main()
{
   try
   {
     std::cout << boost::format("%|+| %2% %1%") % 99 % 100 << std::endl; //!!!
   }
   catch (boost::io::format_error &ex)
   {
     std::cout << ex.what() << std::endl;
   }
}

출력:
boost::bad_format_string: format-string is ill-formed

대신 다음처럼 사용해야 한다.
  ...
std::cout << boost::format("%|+| %|| %||") % 99 % 100 % 99 << std::endl;
혹은
std::cout << boost::format("%|1$+| %|2$| %|1$|") % 99 % 100  << std::endl;

참고로 %|spec| 형태에서 데이터에 대해서 처리할수 없는 flag를 지정하는 경우, 즉
%|+| 는 양의 부호를 붙이라는 의미이고, 숫자인 경우에만 처리가 된다.
만약 문자열에 대해서 지정하면, 조용히 무시된다.
std::cout << boost::format("%|+|") % "AAA" << std::endl; // AAA 출력
