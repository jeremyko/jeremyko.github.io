폴더를 지워야 되는데 too long path 오류가 나면서 안되는 경우. 
최대 경로 길이를 벗어나게 폴더 구조가 생성된 경우인데, 이때 robocopy 유틸을 사용하면 해결 가능하다.

명령 옵션중 /purge 를 활용하는 것인데, 이옵션은 src 경로와 dest 경로의 파일을 비교해서 
src경로에 없는 내용은 delete 시켜주기 때문이다.

예를 들어서 현 위치에서 ./SubDir/aaa/bbb/ccc/..... 라는 구조로 폴더가 존재하는데 
aaa 폴더가 delete 안되는 상황이면, 현위치에서 동일한 이름으로 최상위 aaa  임시 빈 폴더를 
생성 후 다음 명령을 수행하면 ./SubDir/aaa/ 폴더 안의 모든 내용이 없어지게 된다. 
이후 ./SubDir/aaa 폴더만 직접 삭제하면 된다.

robocopy  aaa  ./SubDir/aaa  /purge
